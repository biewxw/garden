BEM é uma metodologia que utiliza um padrão de nomenclaturas para que nosso código seja melhor organizado e lido.

A idéia básica parte de três tipos de estruturas pré-definidas, são elas:

- **Block**: Um composto de elementos. Em outras palavras podemos defini-lo como um "wrapper", ou seja, uma parte do nosso código que servirá como a "base" para o nosso componentes.
- **Element**: Essa estrutura pode ser identificada simplesmente como um item único do qual compõe o resto do nosso bloco, podendo ser um item de uma lista, um título, etc...
- **Modifier**: Modificadores são partes da nossa estrutura que não irão existir sempre. Tal estrutura parte de uma necessidade de se existir diferentes estados para uma estrutura/elemento, por isso são chamados de modificadores. Utilizamos-o por exemplo para definir um estado de estilo para quando um elemento estiver desabilitado na tela, ou com algum tipo de highlight.

Essa estrutura pode ser aplicada por exemplo a uma simples lista de um feed, onde cada item representa uma publicação. Prática:

```html
<ul class="list">
  <li class="list__item list__item--highlight">
    <h2 class="list__title">My publication</h2>
    <p class="list__author>Julio Lozovei</p>
    <p class="list__text">A publication using BEM.</p>
  </li>
</ul>
```

Podemos claramente identificar os três tipos de estrutura nesse código. São eles:

- `.list`: Nosso bloco principal;</br>
- `.list__item`: Um elemento do qual compõe nosso bloco;</br>
- `.list__item--highlight`: Um modificador que cria um estado alternativo para o nosso `.list__item`;
